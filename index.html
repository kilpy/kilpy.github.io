Multi-Agent Genetic Algorithm Combat Sim

This project explores emergent behavior in a custom multi-agent environment using genetic algorithms. Agents are governed by neural networks that dictate their movement, attack decisions, and resource gathering. Through natural selection, the population organically discovers optimal balances between finding food to survive and engaging in combat to eliminate competition.

Final evolved population demonstrating complex combat and evasion strategies.

Early generation baseline behaviors.

Emergent grouping and targeted attacks after 100+ generations.

Evolutionary Mechanics

The fitness function heavily prioritizes survival time, but also rewards agents for dealing damage. This dual-reward system prevents static behaviors where agents just run away. We observe clear evolutionary epochs where different strategies dominate, including "gatherers" who prioritize food, and "predators" who hunt other agents.

Technical Architecture

Computational Fluid Dynamics Surrogate Transformer

Solving the Navier-Stokes equations for Computational Fluid Dynamics (CFD) using traditional mesh generation is notoriously computationally expensive, scaling extremely poorly with required resolution. This project introduces a Transformer-based surrogate model designed for next-frame and autoregressive forecasting of turbulent fluid properties (velocities u, v and pressure p) using multivariate regression in drastically lower dimensional representations.

Autoregressive deployment: Autoregressive fluid density and velocity heatmaps rendering over an internal cylinder structure.

Architectural Highlights (Author Notes)

  • Positional Encoding Intervention: Expand on why standard APE/ALiBi failed physically, and detail your Reynolds-Positional-Representation (ReyPR) architecture.
  • Parameter Efficiency: Discuss how projecting out of the decoder space triggered a parameter explosion (7.8M parameters), and how you mitigated this using a Low-Rank Approximation (LRA) bottleneck rather than lossy mean-pooling.
  • Performance Metrics: Highlight the Pareto-Optimality of the model, specifically how it balanced Mean Squared Error against variance stability over baselines.
  • Physics-Informed Bounds: Add a note on the custom loss function and how it forced the network to respect fluid flow mechanics (Navier-Stokes).

System Architecture

SpotAbnb Data Pipeline

SpotAbnb is a context-aware itinerary generator designed to merge a user's musical identity with travel logistics. Rather than building a standard property search, this system cross-references listening history with live concert availability, and subsequently scores nearby lodging options based on distance to the venue and overall property rating.

End-to-end data flow: Extracting taste profiles, fetching live events, clustering lodging, and future computer vision pipelines.

Feature Engineering & Clustering

The core engine relies on an XGBoost model optimized via an custom distance-weighted loss function. By penalizing properties that are a far radius from the Ticketmaster event coordinates, the model learns to heavily favor walkability. KMeans clustering is employed as a preprocessing step to group the Airbnb listings geographically before the gradient boosting regressor ranks the clusters by density and quality.

Future Work: Visual Vibe Extraction (Dino v3)

Current property scores rely on textual reviews and standard API features (price, beds, star rating). These visual "vibe embeddings" will then become high-impact features within the XGBoost pipeline, allowing the model to recommend lodgings that visually match the auditory aesthetic of the user's Spotify profile.
